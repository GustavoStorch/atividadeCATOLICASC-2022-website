Circuitos

Este circuito está simulando um sinaleiro a qual possui três leds com as referentes cores de um sinaleiro, onde primeiramente etará ligado o led verde (sinaleiro aberto), após um tempo Ele apagará e acenderá o led amarelo (atenção) e após o led amarelo se apaga e acende o vermelho (sinaleiro fechado) e assim sucessivamente se dará o circuto.

Este circuito está simulando uma luz de emergência, onde quando a luz estiver baixa o led vermelho irá se acender, mas enquanto houver uma certa quantidade de luminosidade o led vermelho se manterá acesso. Para isso é utilizado um sensor de fotoressistor.

Este circuito está simulando um motor que irá girar para duas direções: Para frente e para trás, para isso foram usados componentes como uma pilha, o motor, ponte H e um interruptor que quando deslizado ele fará o motor girar para o outro lado.

Este circuito está simulando um circuito com dois motores onde terá 4 direções a qual eles irão girar, os dois para frente, os dois para trás, um pra frente outro pra trás e por fim um prá trás e o outro para frente. Assim sendo possível simular basicamente as rodas de um robo seguidor de linha onde terá os motores a qual terá que girar para estas direções para conseguir realizar as curvas do circuito.

Robô seguidor de linha

Esta é a primeira parte do código do robô seguidor de linha, nesta parte é declarada todas as variáveis globais do projeto e as portas de entrada e saídas de cada componente do robô, podemos ver a declaração das portas do motor, e dos sensores, é definido algumas velocidades para o robô e por fim os valores do pid.

Nesta parte do código temos em vista a função de setup onde são setadas algumas entradas de dados, e seus INPUTS e OUTPUTS, que no caso do nosso robô não há nada para ser declarado nessa função, a não ser o serial begin para os leds que foram retirados do robô.

Logo abaixo temos a função do loop, onde é declarada a função a ser utilizada para que o robô começe a andar.

A função do motorControl é a que diz para qual lado o motor deve girar utilizando os valores recebidos do pid utilizando HIHG e LOW para desligar os motores e mudar a direção de rotação de tais.

A função motorOption é uma pré definição do que o motor deve fazer utilizando da função motorControl, assim ele pode ir pra frente, trás, direita, esquerda e parar caso saia da linha.

A função motorStop é responsável por fazer o robô seguidor de linha parar após um determinado tempo definido, que no nosso caso foi definida em 18500 segundos.

Esta é a função mais importante do robô, nela são declaradas todas as possivéis combinações de valores dos sensores com base na linha a qual ele deve seguir, e para cada combinação é gerado um valor de erro para que possa ser realizado o cálculo de pid e Ele consiga retornar a sua trajetória normal e estável.

A função calculaPid e a que irá realizar o cálculo com base no erro que o seguidor de linha apresentar (O quanto ele sair da linha), assim com esse cálculo teremos exatamente a quantidade de curva que deverá ser realizada pelo robô para que ele retorne a linha, além de manter-ló mais estável na pista.

Já a função de motorPid irá dizer para qual o lado que os motores irão realizar os seus giros.
